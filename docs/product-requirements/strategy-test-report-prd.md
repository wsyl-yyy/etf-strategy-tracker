# ETF 量化策略追踪系统产品需求文档

版本：1.0

日期：2026-05-18

角色：产品经理

交付对象：开发团队、测试团队

依据材料：

- `test-reports/strategy-test-report.html`，测试报告生成时间为 2026-05-18 20:46:41。
- `scripts/strategy_test_mapping.json`。
- `ETF量化交易策略 V3.2 正式执行版.md`。
- 当前实现：`src/etf_tracker/`、`docs/index.html`、`cloudflare-worker/worker.js`。

## 1. 背景与结论

当前系统已经具备基础闭环：GitHub Actions 生成加密日报，GitHub Pages 展示本地解密页面，Cloudflare Worker + KV 保存手机回填成交，页面支持提交、编辑、删除成交。测试报告显示，系统已经覆盖标的代码、A500 实际网格基准价、H2 85% 禁止新增买入、H5 跌破下沿提醒、科创50第一笔触发提示、科创50净值低于1500动作、基础成交 CRUD、以及“不连接券商、不自动交易”的安全边界。

但系统尚未完整落地策略 V3.2。测试报告中，规则映射统计为：

- 通过：11 项。
- 失败：24 项。
- 未覆盖：17 项。
- 人工复核/不适用：4 项。
- 环境阻塞：Python 策略测试因本机 `.venv` 指向缺失的 Python 3.11 未运行。

产品目标不是重做系统，而是在现有简单架构上，把测试人员标记为失败和未覆盖的策略条款，转化为可计算、可展示、可记录、可测试的产品能力。

本系统仍只做记录、复盘和提醒，不接券商接口，不自动下单。

## 2. 产品目标

1. 修复测试报告中所有 `fail` 项，除非产品明确改判为人工复核或不适用。
2. 为测试报告中所有 `uncovered` 项补齐自动化测试证据，除非产品明确改判为人工复核。
3. 让每个策略动作都能追溯到行情、配置、成交记录、资金池状态和风险闸门结论。
4. 保持现有架构：Python 策略评估、静态 Pages 页面、Cloudflare Worker + KV 成交存储。
5. 强化审计能力，让用户能解释“为什么允许买入、为什么禁止买入、为什么需要人工复核”。

## 3. 非目标

- 不做券商登录、券商 API、自动委托或自动成交。
- 不把明文持仓、明文成交、报告密码或提交密码写入公开仓库。
- 不用系统替代季度复核、估值源选择、截图存档、外部市场判断等人工动作。
- 不引入前端框架重做页面。
- 不改动策略 V3.2 之外的交易规则。
- 不增加与当前策略无关的产品功能。

## 4. 产品假设与决策

1. 当策略 V3.2 与当前代码不一致时，以策略 V3.2 为准。
2. “禁止买入”表示日报不得给出可执行买入动作。若用户现实中仍发生了交易，页面可以记录事实，但必须提示违规并要求复盘备注。
3. 估值相关规则必须有固定口径的数据源。自动估值源未确定前，系统应支持人工录入估值值、估值口径和查询时点；缺失估值时，不得默认通过估值条件。
4. A500 实际网格间距默认使用策略建议下限 5.5%，除非季度复核后在配置中填写新的确认值；当前 4% 只能作为历史实现问题，不再作为实际网格默认值。
5. 旧成交记录必须兼容。旧字段 `date` 在没有更细字段时，同时视为“信号确认日”和“实际执行日”。
6. Python 策略测试必须在有效 Python 3.11 环境中重新跑通，不能因为本机 `.venv` 损坏而跳过发布验证。

## 5. 用户与使用场景

策略使用者：

- 每个交易日收盘后打开加密日报。
- 查看是否有买入、卖出、暂停、清仓、复核或禁止动作。
- 手机回填实际成交。
- 修正错误成交。
- 在季度复核时回看每笔交易的触发规则、风险闸门和复盘备注。

测试人员：

- 根据策略测试报告检查每条策略条款是否有实现和测试证据。
- 运行 Python、Worker、报告生成器测试。
- 验证新规则不会破坏已通过项。

开发人员：

- 在现有 Python、Worker、静态页面中补齐需求。
- 保持旧数据兼容。
- 更新测试映射和客户可读测试报告。

## 7. 需求明细

### PRD-001 策略会计口径

系统必须按策略 V3.2 计算投入成本和浮亏比例。

需求：

- 已投入成本按实际买入成交金额计算，不包含交易费用。
- 交易费用独立记录和展示。
- 总持仓浮亏比例按非负数展示：`max(0, 已投入成本 - 当前市值) / 已投入成本`。
- 持仓盈利或不亏损时，总持仓浮亏比例显示为 `0.00%`。
- 风险闸门使用非负浮亏比例，不使用当前代码中的负号表达。

验收标准：

- 买入金额 1000、费用 1 时，投入成本增加 1000，而不是 1001。
- 当前市值 900、投入成本 1000 时，浮亏比例为 10%。
- 当前市值 1100、投入成本 1000 时，浮亏比例为 0%。
- A500 底仓成交均价逻辑不能被破坏。

追踪规则：

- `INVESTED-COST-FEE-MISMATCH`
- `FLOATING-LOSS-RATIO`

### PRD-002 资金池与备用金边界

系统必须保持三个资金池独立。

需求：

- 总资金固定为 10000 元。
- A500 常规网格池为 5000 元。
- 科创50波段池为 2000 元。
- 全局备用金为 3000 元，分为3份，每份1000元。
- A500 卖出收益只回到 A500 常规网格池。
- 科创50卖出或减仓资金只回到科创50波段现金余额。
- 全局备用金仓位单独记录、单独止盈、单独复核，不混入普通网格或科创50波段。
- 任一时点未回收备用金最多动用 2000 元。
- A500 最多动用 2 份备用金，科创50最多动用 1 份备用金。
- 至少保留 1000 元心理安全垫，除非季度复核人工确认调整。
- 跨池调拨只能作为人工复核事项展示，不能静默发生。

验收标准：

- 科创50卖出后只增加科创50波段现金。
- A500 备用金交易能在报告中与 A500 普通网格交易区分。
- 备用金动用超过上限时，报告禁止或阻断候选动作。
- 测试覆盖资金池归属、卖出回池、备用金上限和心理安全垫。

追踪规则：

- `CAPITAL-10000`
- `FUNDS-POOLS`
- `POOL-INDEPENDENCE`
- `RESERVE-SAFETY-CUSHION`
- `A500-RESERVE-LIMITS`
- `KC50-RESERVE-RECOVERY`

### PRD-003 总风险闸门

当总投入成本达到 7000 元且低于 8500 元时，任何新增买入候选都必须经过总风险闸门。

通过标准：

- 2项硬条件全部通过。
- 3项参考条件至少通过2项。

硬条件：

- 总持仓浮亏比例小于 10%。
- A500 和科创50中，至少有一个不处于弱势状态。

弱势定义：

- 当日收盘价低于20日均线。
- 且当日20日均线低于前一交易日20日均线。

参考条件：

- A500估值分位处于近3年合理或偏低区间。
- 科创50估值分位处于近3年合理或偏低区间。
- 距离最近一次止损或止盈不少于10个交易日。

报告要求：

- 展示每个检查项的当前值和是否通过。
- 展示总结果：允许新增买入，或不允许新增买入。
- 当买入信号存在但闸门不通过时，该信号必须显示为禁止或复核，不得显示为可执行买入。
- 估值缺失时，估值条件不得默认为通过。
- 总投入成本达到 8500 元后，无论闸门是否通过，均禁止新增买入。

验收标准：

- 总投入成本 69.99% 时，不强制总风险闸门。
- 总投入成本 70% 时，买入候选带有风险闸门结论。
- 总投入成本 85% 时，所有新增买入均禁止。
- 双标的同时弱势时，硬条件2不通过。
- 测试覆盖全通过、参考条件不足、硬条件失败、估值缺失、85%绝对上限。

追踪规则：

- `H1-RISK-GATE`
- `H2-85-BLOCK`
- `RISK-GATE-STANDARD`
- `TREND-GATE`

### PRD-004 成交记录字段与审计

系统必须支持策略 V3.2 的完整执行记录表。

必填或可维护字段：

- 信号确认日。
- 实际执行日。
- 标的。
- 方向。
- 策略模块。
- 触发规则。
- 执行价格。
- 执行金额。
- 执行份额。
- 交易费用。
- 现金余额。
- 是否触发风险闸门。
- 风险闸门快照。
- 复盘备注。

兼容要求：

- 继续接受旧字段 `date`。
- 旧记录没有信号确认日和实际执行日时，`date` 同时映射为两者。
- Worker、页面、Python 解析器都必须兼容旧记录。

UI/API 要求：

- 手机提交和管理页面能维护新增字段。
- Worker normalization 必须保留新增字段。
- Python trade parser 支持新旧字段别名。
- 报告能展示足够信息，用于重建策略状态和解释信号。

验收标准：

- 带有 `signal_date` 和 `execution_date` 的交易能完整经过页面、Worker、KV、Python 解析。
- 触发规则如 `A500第2格补仓` 能用于后续配对卖出和复核。
- 风险闸门触发时，成交记录能保存闸门结论或快照。
- 旧 CSV 样例和旧 Worker payload 仍可解析。

追踪规则：

- `TRADE-RECORD-BASIC`
- `TRADE-RECORD-MISSING`

### PRD-005 A500 常规网格

系统必须完整支持 A500 常规网格的买入状态和信号。

需求：

- A500 正式标的固定为 `563360`。
- 初始底仓金额为 1000 元。
- A500 实际网格基准价为初始底仓成交均价。
- 普通网格每格买入 600 元。
- 普通补仓最多 5 格。
- 实际网格间距默认 5.5%，除非配置中存在季度复核确认值。
- 不能继续用 4% 作为实际网格默认间距。
- 当前无持仓时，可以展示建议网格参数，但必须标注为建议，不能当作实际交易状态。
- 网格买入候选必须识别已成交档位，避免重复提示。
- 网格买入候选必须受 H1、H2、H5、S1、A500单边下跌保护限制。

验收标准：

- 基准价 1.000、间距 5.5% 时，第1格买入候选在 0.945 或以下触发。
- 已执行第1格时，不重复提示第1格买入。
- 跌破网格下沿后，不提示普通网格补仓。
- 已有 A500 基准价和下沿提醒测试继续通过。

追踪规则：

- `A500-SYMBOL`
- `A500-BASE-PRICE`
- `A500-GRID-SPACING-MISMATCH`
- `A500-INITIAL-BASE`
- `A500-GRID-BUY`
- `H5-A500-LOWER`

### PRD-006 A500 逐笔网格卖出

系统必须将每笔 A500 普通网格买入与对应卖出条件配对。

需求：

- 每笔 A500 普通网格买入必须能识别档位、成交价、份额和是否已关闭。
- 当最新收盘价较该笔买入成交价上涨一个网格间距时，报告提示卖出该笔补仓对应实际份额。
- 初始底仓不参与普通网格卖出。
- 已记录配对卖出的网格买入不再产生卖出信号。
- 部分卖出时，剩余份额状态必须可追踪。

验收标准：

- 0.900 买入、5.5% 间距时，收盘价达到或超过 0.950 后提示卖出。
- 初始底仓不产生普通网格卖出信号。
- 卖出记录存在后，对应买入不再显示为开放补仓。
- 测试覆盖多笔开放买入、一笔触发卖出、已关闭配对不重复提示。

追踪规则：

- `A500-GRID-SELL`

### PRD-007 A500 底仓趋势止盈

系统必须实现 A500 底仓独立止盈规则。

需求：

- 底仓盈利达到 15% 时，提示卖出底仓 1/2。
- 第一次底仓止盈后，追踪剩余底仓的阶段最高收盘价。
- 剩余底仓从阶段最高价回撤 8% 时，提示卖出剩余底仓。
- 阶段最高价和第一次止盈状态必须能跨日报延续。

验收标准：

- 盈利达到 15% 时出现第一次底仓止盈动作。
- 已记录第一次止盈后，不重复提示同一动作。
- 第一次止盈后出现更高收盘价时，更新阶段高点。
- 从阶段高点回撤 8% 时，提示卖出剩余底仓。

追踪规则：

- `A500-BASE-PROFIT-15`
- `A500-BASE-TRAIL-8`

### PRD-008 A500 单边保护

系统必须识别 A500 单边下跌、恢复和单边上涨保护。

单边下跌暂停条件：

- 连续3格补仓成交且无任何一格触发卖出。
- 或价格连续10个交易日低于网格下沿。

恢复条件：

- 价格重新回到网格区间内。
- 且连续5个交易日维持在区间内。

单边上涨：

- 保留现有连续10日高于网格上沿提醒。
- 不追买。
- 底仓趋势止盈满足时优先提示止盈。

验收标准：

- 3笔开放补仓且无卖出时，暂停普通网格补仓。
- 连续10日低于下沿时，暂停普通网格补仓。
- 连续4日回到区间内不得恢复，连续5日才恢复。
- 已有上沿提醒测试继续通过。

追踪规则：

- `A500-DOWNTREND-PROTECT`
- `A500-UPTREND-PROTECT`

### PRD-009 A500 备用金规则

系统必须把 A500 备用金作为人工复核候选和独立仓位管理。

候选条件：

- A组：跌破网格下沿后继续下跌 7%-8%，且估值低于历史20%分位。
- B组：估值低于历史10%分位。
- C组：跌破网格下沿后继续下跌10%以上。

动用约束：

- 每次最多动用 1000 元。
- 默认两次备用金动用之间至少间隔20个交易日。
- 如果间隔期内价格反弹超过5%，随后再次跌破前低，且估值仍低于20%分位，可缩短至10个交易日。
- 备用金动作始终是人工复核候选，不自动买入。

仓位管理：

- 盈利10%-15%时，分2次卖出锁定收益。
- 亏损超过8%时，暂停下一笔备用金投入。
- 季度复核时仍亏损超过12%时，提示认亏退出、降级长持或继续持有的人工决策。

验收标准：

- 报告区分 A500 普通仓位和 A500 备用金仓位。
- 备用金候选展示命中的条件组和人工复核原因。
- 缺失估值时，A组和B组不能通过。
- 测试覆盖动用间隔、缩短间隔、盈利止盈、亏损复核。

追踪规则：

- `A500-RESERVE-MANUAL`
- `A500-RESERVE-LIMITS`
- `VALUATION-SOURCE`

### PRD-010 科创50买入阶梯与估值交叉验证

系统必须完整支持科创50四笔买入规则。

需求：

- 科创50正式标的固定为 `588000`。
- 第一笔买入信号触发日向前回看12个月收盘价最高值，并锁定为本轮阶段高点。
- 第一笔：阶段高点回撤22%，目标金额400元。
- 第二笔：阶段高点回撤30%，目标金额500元。
- 第三笔：阶段高点回撤38%，目标金额500元。
- 第四笔：阶段高点回撤45%，目标金额600元，且必须满足第4笔过滤条件。
- 第一笔22%触发时，如果科创50估值仍高于历史50%分位，不执行第一笔。
- 延后后，回撤达到25%或估值回落至50%分位及以下，二者满足其一即可重新触发第一笔候选。
- 第一笔实际触发后，第二至第四笔继续使用锁定阶段高点，不重新调整。
- 买入候选必须受总风险闸门、H2、H3、H4、S1、科创50分级风控限制。

验收标准：

- 已买第一笔后，30%回撤可以触发第二笔候选。
- 已买前两笔后，38%回撤可以触发第三笔候选。
- 22%回撤但估值高于50%分位时，只提示延后或复核，不提示可执行买入。
- 延后后达到25%回撤时，可以重新触发第一笔候选，前提是不被硬规则禁止。
- 测试覆盖累计投入边界。

追踪规则：

- `KC50-SYMBOL`
- `KC50-FIRST-BUY`
- `KC50-SECOND-THIRD-BUY`
- `KC50-VALUATION-CROSS`

### PRD-011 科创50第4笔过滤

科创50第4笔不得仅因45%回撤机械触发。

需求：

- 必须已有第三笔买入记录。
- 第三笔后，连续5个交易日不创20日新低。
- 且其中至少某一日收盘价站上5日均线。

- 估值缺失时，显示人工复核，不得显示参考条件通过。

验收标准：

- 未买第三笔时，45%回撤不触发第四笔。
- 第三笔后少于5个不创新低交易日，不触发第四笔。
- 第5个不创新低交易日仍未站上5日均线时，继续等待。
- 两项过滤条件都满足后，若无硬规则禁止，可以显示第四笔候选。

追踪规则：

- `KC50-FOURTH-BUY`
- `NEW-LOW`

### PRD-012 科创50份额和目标金额校验

验收标准：

- 第一笔买入超过400元被拒绝或标记。

### PRD-013 科创50固定止盈与移动止盈

系统必须实现科创50止盈规则。

需求：

- 止盈基准为首次止盈前锁定的综合持仓成本。
- 盈利15%时，提示卖出1/3仓位。
- 盈利30%时，提示再卖出1/3仓位。
- 对应卖出已记录后，不重复提示同一固定止盈动作。
- 第二次固定止盈后，剩余1/3仓位进入移动止盈。
- 第二次止盈成交日收盘价作为移动止盈初始阶段高点。
- 后续更高收盘价更新阶段高点。
- 从阶段高点回撤12%时，提示卖出剩余仓位。
- 连续2个交易日收盘价跌破10日均线时，提示卖出剩余仓位。
- S2 高波动阶段时，移动止盈回撤参数从12%放宽到13%；波动率回落到50%及以下后恢复12%。

验收标准：

- 盈利15%触发第一次固定止盈。
- 盈利30%在第一次止盈状态满足后触发第二次固定止盈。
- 新高会更新移动止盈阶段高点。
- 根据 S2 状态，12%或13%回撤触发移动止盈。
- 连续2日跌破10日均线触发移动止盈。

追踪规则：

- `KC50-PROFIT-TAKE`
- `KC50-TRAILING-PROFIT`
- `S2-VOLATILITY`

### PRD-014 科创50净值止损、暂停与恢复

系统必须完整实现科创50波段账户净值状态。

需求：

- 科创50波段账户净值 = 科创50持仓市值 + 科创50波段现金余额，不包含全局备用金。
- 净值低于1700元时，停止新增买入，进入观察。
- 净值低于1500元时，提示卖出当前科创50仓位的1/2，并暂停全部新增买入。
- 净值低于1400元时，提示清仓全部剩余科创50仓位。
- H3 减仓后，即使净值回到1500元以上，也不能自动恢复买入资格。
- 恢复买入资格必须满足净值重新回到1800元以上，并维持5个交易日。

验收标准：

- 低于1700元产生停止新增买入提醒。
- 低于1500元产生减仓1/2动作和暂停状态。
- 低于1400元产生清仓动作。
- 连续4日高于1800元不得恢复，连续5日才恢复。
- 测试覆盖1400和1500阈值同时命中时的优先级。

追踪规则：

- `H3-NET-VALUE`
- `KC50-NET-1700`
- `KC50-NET-1500`
- `KC50-NET-1400`
- `KC50-RECOVER-1800`

### PRD-015 科创50分级风控

系统必须按买入笔数应用科创50分级风控。

需求：

- 买入1笔：无额外风控，仅按标准流程跟踪。
- 买入2笔：连续10个交易日收盘价低于20日均线时，暂停新增买入。
- 买入3笔：连续15个交易日低于下行的20日均线时，第4笔必须等待第4笔过滤条件和人工复核。
- 买满4笔：连续20个交易日低于20日均线、20日均线持续下行、且相对综合持仓成本亏损超过15%时，启动完整风控减仓。
- 完整风控减仓分3次，每周减1/3。
- 减仓过程中，若收盘价重新站上20日均线且连续3个交易日不跌破，则暂停后续减仓，剩余仓位恢复正常止盈规则。
- 优先级为：H3兜底风控 > 固定止盈/移动止盈 > 风控减仓 > 新增买入。

验收标准：

- 测试覆盖2笔、3笔、4笔阶段。
- H3 阈值命中时覆盖计划中的周度风控减仓。
- 连续3日站上20日均线后，后续减仓暂停。

追踪规则：

- `KC50-GRADED-RISK`

### PRD-016 科创50时间复核

系统必须提醒科创50长期持仓复核。

需求：

- 按交易日计算持仓天数。
- 持仓超过60个交易日、盈利5%-15%、且收盘价跌破20日均线时，提示可选择卖出一半或全部并记录理由。
- 持仓超过120个交易日、盈利仍不足5%时，提示强制专项复核。
- 时间复核只产生提醒，不自动交易。

验收标准：

- 59个交易日不触发60日复核。
- 超过60个交易日且满足盈利和均线条件时触发复核。
- 超过120个交易日且盈利不足5%时触发强制复核。

追踪规则：

- `KC50-TIME-REVIEW`

### PRD-017 S1 相关性监控

系统必须实现 S1 软规则的限制和记录。

需求：

- 高度同步时，两标的合计单日新增买入计划金额不得超过800元。
- A500 和科创50同日触发新增买入时，优先 A500，科创50延后1个交易日复核。
- 两标的同时处于补仓或买入状态时，新增投入节奏减半，未执行部分进入次日复核，不自动补买。
- 800元限制按计划下单金额判断，不按最终成交金额判断。
- 用户覆盖 S1 时，成交记录必须写明复盘备注。

验收标准：

- 相关系数0.86时进入 S1 状态。
- A500 600元和科创50 500元同日候选时，触发800元限制并优先 A500。
- 延后后的科创50候选在延后日重新检查硬规则和 S1 状态。

追踪规则：

- `S1-CORRELATION`
- `CORRELATION`

### PRD-018 S2 波动率自适应

### PRD-019 清仓重启与连续10日浮亏复核

系统必须实现清仓后的重启资格，以及 H7 连续浮亏规则。

清仓重启需求：

- 任一策略清仓后，至少等待5个交易日。
- 重启前必须重新锁定基准价、阶段高点、网格上下沿和资金池余额。
- 重启第一笔买入金额不超过正常首笔的50%，作为试探仓。
- 试探仓有效后，视为本轮第一笔的一部分。
- 恢复正常节奏时，第一笔剩余金额可在下一个交易日补足；若试探仓失效或触发暂停规则，则不补足。

H7需求：

- 每笔新增买入必须能在第10个交易日复核是否仍为浮亏。
- 单标的连续3笔新增买入在第10个交易日复核仍为浮亏，且复核当日3笔全部仍浮亏时，暂停该标的新增买入并提示专项复核。

验收标准：

- 清仓后第4个交易日不允许重启。
- 清仓后第5个交易日可以进入重启候选，但必须完成关键参数重锁。
- 重启试探仓超过正常首笔50%时，提示禁止或违规。
- 连续3笔10日浮亏时，暂停该标的新增买入。

追踪规则：

- `H6-RESTART`
- `H7-THREE-LOSS-BUYS`
- `RESTART-AFTER-LIQUIDATION`

### PRD-020 策略失效与收益锁定

系统必须展示长期策略失效和收益锁定状态。

策略失效需求：

- 科创50连续2轮触发 H3 应急风控时，暂停科创50策略至少30个交易日。
- A500 网格连续90个交易日无有效买卖成交时，提示季度复核是否调整网格间距或暂停策略。
- 总账户最大回撤超过12%时，暂停所有新增买入，只允许止盈、减仓和复核。
- 策略运行6个月后，对比总账户收益率与同期沪深300涨跌幅、货币基金收益中较高者；若明显落后，提示策略有效性复核。

收益锁定需求：

- 总账户累计收益达到初始本金15%时，提示至少提取收益的1/3进入不可交易现金池。
- 不可交易现金池不得用于主动交易。
- 不可交易现金池只有在季度复核或策略重启时才能讨论调配。

验收标准：

- 总账户最大回撤计算包含 ETF 市值、所有资金池现金和不可交易现金池。
- 最大回撤超过12%时，所有新增买入候选被禁止。
- 累计收益达到15%时，报告提示收益锁定。
- 已锁定收益显示为不可交易资金，不能用于新增买入。

追踪规则：

- `STRATEGY-INVALIDATION`
- `PROFIT-LOCK`

## 8. 报告展示要求

加密日报必须展示：

- 报告生成时间和最新行情日期。
- 行情数据是否为当日或最新交易日。

- 交易记录字段缺失提醒。

文案要求：

- 当系统有足够数据判断时，不使用模糊表达。

优先从成交记录和行情历史推导状态，只有无法稳定推导时才增加显式状态字段。

## 11. 人工复核边界

以下内容保留人工复核，不要求系统自动决策：

- 盘中条件单、限价委托和实际下单。
- 估值平台选择、截图或数据存档方式。
- 季度复核的主观结论。
- A500备用金是否实际动用。
- 科创50备用金回暖的外部市场软信号。
- 策略运行6个月后是否“明显落后”基准的最终判断。
- 任何突破资金池和备用金边界的例外。

## 13. 开发前需确认的问题

1. Worker 对违规真实成交的处理方式。产品建议：允许保存真实成交，但强提示违规并要求复盘备注；日报不得把该成交视为合规执行。

4. “策略运行6个月后显著落后基准”的量化阈值。产品建议：首期只展示对比结果并要求人工复核，不自动判定失效。
